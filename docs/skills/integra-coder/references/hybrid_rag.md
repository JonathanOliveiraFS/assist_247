# Hybrid RAG Pattern (ChromaDB + BM25)

## Overview
Integra.ai uses a hybrid retrieval strategy to combine semantic understanding with keyword precision. This is implemented via LangChain's `EnsembleRetriever`.

## Key Components

### 1. Multi-tenancy Isolation
Always use the `customer_id` (tenant ID) to isolate data.
- **ChromaDB:** Use separate collections named `collection_{customer_id}`.
- **BM25:** Load/Save separate index files (`.pkl`) per customer.

### 2. Implementation Pattern
```python
from langchain.retrievers import EnsembleRetriever
from langchain_community.retrievers import BM25Retriever
from langchain_chroma import Chroma

def get_hybrid_retriever(customer_id, embeddings, persist_directory, bm25_path):
    # 1. Semantic Retriever (Chroma)
    vectorstore = Chroma(
        persist_directory=persist_directory,
        collection_name=f"cliente_{customer_id}",
        embedding_function=embeddings
    )
    semantic_retriever = vectorstore.as_retriever(search_kwargs={"k": 4})

    # 2. Lexical Retriever (BM25)
    # Assumes BM25 index is pre-generated by Kestra/ETL
    bm25_retriever = BM25Retriever.load_local(bm25_path)
    bm25_retriever.k = 4

    # 3. Ensemble
    ensemble_retriever = EnsembleRetriever(
        retrievers=[semantic_retriever, bm25_retriever],
        weights=[0.6, 0.4]  # Semantic favored for context, BM25 for terms
    )
    return ensemble_retriever
```

## Best Practices
- **Weights:** Keep semantic weight higher (0.6-0.7) for conversation, but adjust to 0.5/0.5 for technical procedural documents.
- **Preprocessing:** Ensure queries are cleaned of WhatsApp-specific noise (emojis, multiple spaces) before retrieval.
